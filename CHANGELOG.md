# Changelog

本文件按 Keep a Changelog 风格记录用户可见变化。

## [Unreleased]

- **云端盈亏复盘完整快照**: 根入口页补齐本地盈亏复盘的亏损与盈利统计、ATR 热力矩阵、原因分类和 20 笔首屏订单数据，避免线上只显示缩略模块。

- **实时信号刷新隔离**: 实时交易监控刷新不再改写交易复盘当前标的，避免成交记录被空结果覆盖。

- **实时交易最新行情**: 「原因与证据」按标的展示最新分钟 K 的收盘、OHLC、成交量和 bars 数，便于核对实时信号依据。

- **热力矩阵缺证据行**: 复盘热力时间矩阵在没有缺 ATR 证据订单时会隐藏缺证据行，减少空行干扰。

- **数据下钻收益汇总**: 「数据下钻」热力时间矩阵在时间窗口底部新增收益合计，便于横向比较每个日内窗口的总收益。

- **盈亏复盘切换**: 下钻复盘中的「亏损复盘」改为「盈亏复盘」，默认仅看亏损单；切到仅看盈利单时，热力时间矩阵和订单明细展示盈利单，原因模块保持为空。

- **云端盈亏复盘一致性**: 根入口页补齐本地盈亏复盘结构，支持仅看盈利单/仅看亏损单切换，并同步热力矩阵、原因分类和订单列表的默认展示语义。

- **云端亏损复盘默认视图**: 根入口页默认打开亏损复盘，使热力图及下方原因分类、亏损单列表与本地复盘台截图一致；数据下钻仍可切换查看。

- **云端根入口样式一致性**: 根入口页改为与本地复盘台一致的三工作区导航和全宽复盘面板，数据下钻与亏损复盘不再使用旧版左侧日期列表布局。

- **根入口亏损视图深链**: 根入口页支持直接打开亏损复盘视图，减少刷新后停留在默认数据下钻导致的误判。

- **根入口亏损复盘**: GitHub Pages 根入口页支持在「数据下钻」和「亏损复盘」两个静态模块之间切换，便于不启动本地服务时查看当前复盘口径。

- **最大回撤显示**: 成交记录最大回撤列只展示回撤金额，不再混入分钟线数量或每股回撤，避免把金额尾数误读成 bars 数量。

- **数据下钻时间矩阵**: 「数据下钻」热力时间矩阵改为只读展示全部订单分布，并在摘要中标出最大盈利区和最大亏损区；日期/标的下钻列表只受时间范围影响。

- **ATR 波动分层**: 复盘热力时间矩阵纵轴改用开仓 1min K 振幅 / 前 20 根 ATR 的倍数，分为极端冲击、高波动、常规波动、低波动和缺 ATR 证据，不再用美元亏损或回撤阈值判断波动环境。

- **亏损热力时间矩阵**: 「亏损复盘」热力时间矩阵改为只读展示，并在摘要中标出最大亏损区；订单明细继续由时间范围和原因筛选控制。

- **亏损复盘时间筛选**: 「亏损复盘」可按全部、本月、本周或特定时间段统一筛选统计、原因汇总和订单明细，统计指标整理为一行展示。

- **亏损复盘独立视图**: 「亏损复盘」tab 仅展示亏损复盘内容，新增一级/二级原因饼图联动筛选、亏损单排序和 20 笔分页。

- **错误提示可读化**: 表单校验、缺字段和后端业务拒绝会显示需要补充或修正的操作，不再把接口状态码直接展示给用户。

- **STP 导入行情补缺**: 上传 STP TXT 后会按本批成交的日期和标的检查本地分钟线，缺失时补写行情归档；成交事实仍只来自 STP TXT。

- **亏损复盘下钻**: 下钻复盘新增「亏损复盘」tab，集中展示全部亏损交易组的原因分类汇总和明细入口。

- **亏损筛选图表联动**: 勾选「仅看亏损单」时，上方分钟蜡烛图同步只显示这些亏损交易组对应的买卖点。

- **Trade Replay EMA20**: Trade Replay 弹层蜡烛图新增 EMA20 辅助线，只用于本地归档分钟线的视觉复盘，不改写成交事实或策略证据。

- **Trade Replay 弹层交互**: Trade Replay 滚动时头部保持固定，点击弹层外阴影区域可关闭弹层。

- **重登录前端校验**: 本地登录入口会通过前端代理校验后端 OpenAPI 合同，并在进程短暂可用后复查，避免打开已经失效或指向旧后端的 fallback 页面。

- **成交记录亏损筛选**: 成交记录新增「仅看亏损单」勾选项，可只查看当前范围内 closed 且 PnL 小于 0 的交易组，过滤不改写复盘事实。

- **亏损交易复盘**: 成交记录不再显示单独「复盘」操作按钮，亏损原因改为在 Trade Replay 订单明细下方维护，复盘结论不会改写成交事实。

- **分钟蜡烛复盘留白**: 分钟蜡烛图会按成交记录查看弹层同一口径，在首尾各多显示 10 根已归档分钟线，买卖点和信号不再贴住图表边缘。

- **交易复盘离线行情**: 交易复盘和 Replay 弹层只刷新本地已保存分钟线归档，不再在打开成交回放时自动触发行情拉取。

- **策略测试自然日窗口**: 测试复盘改为读取截止日前最近 30 天（自然日）内的本地归档，不再向更早交易日补足 30 个交易日。

- **登录入口版本校验**: 本地登录入口会校验实时交易前端指纹并带版本参数打开页面，避免重登后继续看到旧实时交易模块。

- **实时交易信号面板**: 「下单信号」模块展开后端 `signals[]`，同时展示真实 BUY/SELL 订单里的开仓和关仓明细；开仓单显示按资金模型计算的股数和止损止盈，关仓单显示平仓股数与原因标签且不再显示止损止盈，HOLD、失败、状态、完整策略动作、版本、provider、原因码和 hash 证据保留在「原因与证据」模块。

- **实时交易监控**: 「实时交易」标的选择支持下拉多选，行情源默认 Yahoo；开启监控后每 30 秒读取最新行情并按标的展示只读信号、原因和证据。

- **多标的策略优化**: 策略优化会按输入标的组生成全局综合最优候选，并显示覆盖范围和模板版本，避免旧版本或单标的局部结果误导参数选择。
- **策略命名**: 区间边缘反转策略在策略下拉和配置中显示为「PA-1min边缘狙击反转策略v1.1」，下拉选项不再追加内部模板 key。
- **实时交易行情源**: 「实时交易」行情源新增 Yahoo 实时分钟线，并在策略下拉和信号结果展示最新策略版本号。
- **策略资金PNL**: 策略配置新增初始本金和入场资金比例，单日复盘、最近30天测试和优化候选会按后端资金模型展示 PnL。
- **实时交易信号预览**: 复盘台新增「实时交易」tab，可选择策略和标的，用实时行情生成只读 BUY/SELL/HOLD 信号、下单原因和证据 hash；策略未开启或行情不可用时会明确显示不可下单状态。
- **下钻复盘日数据**: 下钻复盘的日统计改为只展示订单数、股数和 PnL，减少与当前复盘指标的重复信息。
- **策略汇总下钻布局**: 策略测试汇总下钻面板改为按自身内容顶部对齐，避免右侧单日复盘图表把左侧列表拉出大面积空白。
- **策略订单对收益**: 策略复盘的每个开平仓订单组现在直接显示后端 run read model 计算的 PnL，未闭合或缺少收益证据时显示 N/A。
- **策略历史回退**: 历史版本回退会同时恢复模板版本和参数快照，避免参数已回退但版本仍显示新版、按钮误判不可点击。

- **策略版本回退**: 交易策略配置弹层新增策略版本记录，可查看参数变更证据并一键回退到历史参数快照，历史策略 run 和优化候选不会被覆盖。
- **复盘数据范围提示**: 策略测试面板会按截止日前最近30天自然日窗口显示每个标的已有分钟线覆盖，避免历史失败归档被算入当前30天判断。
- **Range Fader v1.1 回退**: 1 分钟区间边缘狙击策略恢复为中轴分批止盈、保本止损和对侧边缘最终目标，避免继续使用反向 K 追踪版本。
- **本地分钟线归档**: 新增本地研究标的组归档脚本，可把 MU、NVDA、SPY 等 1 分钟线批量写入本地 SQLite 归档，策略测试继续只读已保存分钟线。

- **复盘页加载性能**: 策略 run 列表、交易组详情、复盘摘要和下钻汇总改为轻量首屏读取，策略工作区按需加载，完整指标序列、组内成交和交易评价明细延后到选中详情时读取，复盘首屏已压到 2 秒内。
- **后端版本自检**: 本地启动入口会检查策略 run 详情路由，避免复盘页连到旧后端后继续加载完整指标大包。

- **BB Squeeze ATR 标准化**: 1 分钟布林带收缩突破策略改用 ATR 倍数计算止损和第一目标，配置弹层可调 ATR 周期、止损倍数和目标倍数，减少不同标的波动差异带来的固定盈亏比失真。

- **策略汇总指标**: 多标的整体汇总会按最新测试批次聚合盈亏比和回撤，不再把可计算结果显示为 N/A 或 0.00。

- **策略汇总状态口径**: 日期汇总只展示有策略订单的标的，状态优先来自这些可见标的，避免 0 订单且行情失败的标的把整日误标为行情不可用。

- **策略汇总密度**: 测试复盘的日期和标的汇总卡片改为紧凑指标行，减少重复说明文字，让单日下钻入口更靠前。

- **策略测试复盘层级**: 测试复盘模块先展示策略整体指标，再按日期或标的汇总下钻到单日复盘，多标的研究不再只围绕主标的展开。

- **多标的扫描反馈**: 策略测试页输入标的组后会立即为每个标的显示待运行或最新批次状态，避免只有已有批次变化时才看起来有反馈。

- **多标的策略扫描**: 策略测试页支持输入标的组，按每个标的分别拉取 30 日数据、运行测试和保存优化结果，便于一次比较高流动性科技股的信号质量。

- **下钻复盘预载**: 交易日下钻会提前缓存各日标的分组，切换日期时不再临时停在读取标的状态。

- **当前复盘口径**: 当前复盘指标会跟随选中的交易日和标的读取同一 committed fills 摘要，避免下钻统计有成交而当前复盘卡片显示 0 或 N/A。

- **分钟蜡烛图缩放**: 图表 Y 轴会优先围绕 K 线实体、成交点和策略执行价自适应，孤立异常长影线、辅助指标或目标价不再压扁正常价格区间。

- **Range Fader 模板版本**: 套用优化候选后会把 1 分钟 Range Fader 当前配置回填到最新模板版本，并保留候选来源和配置历史证据。

- **策略优化套用参数**: 套用最佳候选或候选组时不再被 30 日详情预取阻塞，页面会显示套用反馈并保留历史优化证据。

- **策略测试操作**: 策略测试页的策略配置卡片可直接发起单日策略复盘，并在运行中、完成或失败时保留可见反馈。

- **策略复盘口径**: 策略配置弹层区分单日策略 run 与最近 30 天测试复盘，避免 0 信号和聚合信号数混为同一结果。

- **开仓信号明细**: 展开开仓信号后会完整显示开仓和平仓策略动作，右侧只展示信号标签，指标数据改为订单下方可横向展开的明细折叠条。
- **开平仓组图表**: 单组开平仓信号展开后会显示该区间内分钟蜡烛图，并标出该组开仓和平仓信号位置。
- **策略参数套用追溯**: 策略优化候选被显式套用时会同步当前模板版本，并记录历史版本、候选来源和变更原因，避免参数已更新但版本仍显示旧口径。
- **区间边缘狙击策略**: 交易策略配置弹层新增 1 分钟 Range Fader 策略，可在震荡区间上下边缘识别假突破反转，并按中轴磁铁和对侧边缘生成历史复盘信号。
- **策略配置弹层滚动**: 交易策略配置弹层新增稳定的主体滚动区，策略参数和完整策略描述不会再被视口底部截断。
- **策略模板启动自检**: 本地启动入口会要求最新策略模板，避免旧后端被复用后策略配置缺少新策略。
- **趋势中继策略**: 复盘台新增 1 分钟 PA 趋势中继策略，可在强趋势突破后等待 H2/L2 二级回调入场，并用 9 EMA 追踪出场；缺归档或回调不缩量时不生成成功信号。
- **根入口页**: GitHub Pages 根入口改为带数据的复盘台静态快照，打开链接即可看到 P2 Review Desk 关键区域。
- **本地启动诊断**: Windows 启动入口会在后端阶段验证复盘 API，并在数据库被旧后端锁住时显示端口 owner PID，不再误停留在前端启动等待。
- **成交组回撤追溯**: 成交记录和 Replay 弹层新增持仓最大回撤，按已归档分钟线最高/最低价计算，并在缺分钟线或未清仓时显示不可用状态。
- **成交记录追溯入口**: 成交记录里的查看操作会先打开 Replay 弹层，再加载分钟线数据；行情缺失或归档失败不再阻止弹层打开。
- **复盘仪表盘指标**: 仪表盘新增单笔期望值、每股净收益和持仓最大回撤；单日指标聚焦交易质量，不再展示成交数或每股净收益，并让持仓最大回撤保持在同一指标行。
- **日期下钻复盘**: 切换交易日时先等待该日标的分组加载完成，避免把旧数据空态误显示成“该日没有标的”。
- **复盘盈亏比显示**: 全盈利的已平仓复盘会显示无亏损盈亏比，不再误显示为 N/A。
- **分钟蜡烛买卖点对齐**: 成交标记的尖点会锚定实际成交价和对应分钟 K 线，不再因三角形中心偏移造成视觉错位。
- **下钻复盘日期切换**: 按交易日切换时，日统计和标的列表会绑定当前选中日期，不再被旧请求或旧复盘上下文覆盖成空数据。
- **本地启动自愈**: 启动入口遇到旧后端占用默认端口时会自动切到备用端口启动复盘台，不再要求用户先手动关闭旧进程。
- **交易复盘下钻**: 交易复盘顶部展示有记录以来汇总，并支持按交易日或标的进入对应复盘模块。
- **本地启动自检**: 启动入口会检查复盘汇总 API，避免旧后端占用时交易复盘下钻显示 API 404。
- **策略单日复盘图表**: 单日复盘默认显示策略开平仓标记并聚焦信号区间；可勾选展示全天 K 线。
- **策略测试单日预取**: 最近 30 天结果载入后会提前缓存单日图形和策略订单明细，切换日期时不再等到点击后才开始读取。
- **策略优化搜索覆盖**: 默认优化网格扩大到更多参数和步长，并在候选上限内做稳定抽样，提升参数研究覆盖面。
- **策略测试复盘结构**: 30 天测试结果可从日列表直接选择单日，查看该日策略图形和订单明细。
- **动能策略优化**: 默认优化网格会跳过不合法的 ADX 阈值组合，避免单个候选参数组合导致整次优化无法运行。
- **策略优化候选组**: 策略优化区改为更宽的候选组视图，逐组展示候选参数项值、稳定性指标和手动套用入口。
- **策略测试数据反馈**: 拉取最近 30 天自然日数据时会立即显示进度，完成后说明可用天数和首个不可用日期原因。
- **策略测试数据准备**: 策略测试页支持手工输入研究标的，并显式拉取最近 30 天自然日分钟线归档，覆盖不足时可先补数据再运行测试。
- **策略测试工作区**: 复盘台新增独立策略测试 tab，可在策略配置、单日测试、最近 30 天复盘和参数优化之间切换；优化候选只展示研究结果，需用户显式套用才会更新策略配置。
- **动能过滤均值回归策略**: 策略新增 ADX 趋势熔断、ATR 动态止损和 11:30-13:30 午盘窗口，趋势日会生成 0 信号而不是继续触发均值回归。
- **策略信号日内总览**: 策略信号详情弹层新增开平仓期间蜡烛图，集中展示当天所有后端策略动作的实际持仓区间走势。
- **策略信号统计兼容**: 详情弹层在策略 run 已有开平仓信号但汇总字段缺失时，会基于 run read model 展示盈亏、胜率和盈亏比，不再显示 N/A。
- **策略信号详情统计**: 详情弹层新增策略信号盈亏、胜率和盈亏比，并压缩汇总区空白，便于快速判断每次策略 run 的复盘结果。
- **策略信号详情弹层**: 展开后续开平仓组时，弹层会保留内部滚动区并直接展示订单内容，避免明细表被视口裁切。
- **策略信号详情**: 详情弹层按每一组开平仓展示独立蜡烛图和订单明细，帮助复查后端策略 run 的信号证据。
- **动能过滤均值回归策略**: 复盘台新增 1 分钟均值回归策略，可在 QQQ/SMH 同向 VWAP 过滤下等待布林带反转入场，并按中轨部分止盈、对侧外轨最终止盈和保本止损生成历史复盘信号。
- **动能过滤归档**: 启用动能过滤均值回归策略后，分钟线归档会补齐同日 QQQ/SMH 上下文，并在缺上下文时给出可执行提示。
- **本地启动自检**: 本地启动入口会识别缺少最新策略 API 或策略模板的旧后端；只读检查也会要求后端和前端都可访问，避免误报启动成功。
- **策略复盘反馈布局**: 策略配置弹层中的复盘反馈改为紧凑提示，参数配置区保留内部滚动，避免反馈后挤出大面积空白。
- **策略复盘反馈**: 点击“策略复盘”后会立即显示运行中提示，并在后端返回后展示完成、0 信号、缺归档或失败状态，避免等待计算时没有交互反馈。
- **策略配置操作**: 交易策略配置弹层固定展示新增入口，创建策略时可先选择模板并调整参数；编辑当前策略后需先保存，随后才能用最新策略逻辑发起策略复盘。
- **策略复盘状态**: 分钟蜡烛复盘会区分策略未运行、0 个策略信号和非 0 信号，并允许查看 0 信号详情，避免把无信号结果误判为图层异常。
- **Trade Replay 订单明细**: Replay 弹层在蜡烛图下方展示逐笔订单和成交追溯信息，便于核对买卖阶段、价格、数量和证据来源。
- **复盘台布局**: 复盘台改为主图优先的交易工作区，盘前 Watchlist 前移到首屏右侧，导入批次和隔离行沉到底部证据区，Trade Replay 降为成交记录中的查看动作。
- **机构流动性掠夺策略**: 复盘台新增 1 分钟扫损毛刺策略，可配置局部窗口、影线占比和 OCO 历史出场模式，并在缺归档或行情不可用时保持失败状态可见。
- **交易策略配置**: 复盘台可添加、开启并运行 BB Squeeze Breakout 历史策略，分钟蜡烛图会叠加策略指标和开平仓信号；缺归档或行情不可用时只显示状态，不渲染假信号。
- **策略配置结构化**: 交易策略配置弹层按开仓、平仓、止盈止损分组编辑参数，并在右侧展示当前配置生成的完整策略描述。
- **策略配置弹层布局**: 策略配置弹层改为左侧紧凑策略列表、中间参数工作区和右侧规则说明栏，保存、运行和启停操作更清晰。
- **BB Squeeze 策略优化**: 策略新增 9 EMA 出场缓冲、最小绝对带宽过滤和 2:1 被动止盈触达建模，减少过早离场和死寂窄幅假突破。
- **策略配置弹层**: 交易策略配置改为操作按钮打开弹层，复盘台主界面保留策略状态和图表操作。
- **策略信号详情**: 分钟蜡烛图中的策略信号数量可点击查看，每条信号会展示执行动作、价格、止损止盈、触发原因和指标。
- **分钟蜡烛图层开关**: 分钟蜡烛复盘可单独控制 committed 买卖点和策略信号点显示，买卖点默认显示，策略点默认隐藏。
- **API 404 提示**: 复盘台和本地启动入口在检测到旧后端进程占用时，会明确提示重启本项目进程，不再只显示笼统的 API 404。
- **交易组 Replay**: 成交记录现在按每一次开仓到清仓配对展示；点击 Replay 可查看该次交易的分钟蜡烛、成交量、VWAP、高低点和规则化交易评价，未清仓或行情缺失时会明确显示原因。
- **交易组表格**: 成交记录的开仓、清仓和持仓时长改为分行展示，评价与追溯列只保留用户需要的等级、成交笔数和 Replay 操作。
- **Replay 买卖点**: Replay 弹层蜡烛图中的买入、卖出成交点改为更醒目的标签和箭头标记，分钟蜡烛复盘主图的信号样式保持不变。
- **分钟蜡烛复盘**: 复盘页可按日期和标的查看已归档分钟蜡烛图，图表会自动缩放到第一笔到最后一笔成交，并把买入、卖出成交点标注在对应走势上。
- **后端路由自检**: 本地启动入口会识别缺少策略配置能力的旧后端，避免复盘页连到旧服务后只显示 API 404。
- **Yahoo 分钟线离线归档**: 可按已提交成交的交易日和标的归档 Yahoo 分钟线；缺数据或 provider 失败会留下可复查状态，不会改写 STP 成交真相。

### 新增

- **P1 行情上下文与 Watchlist**: 复盘台现在可以为 committed fills 生成可追溯行情 snapshot，并生成带入选原因和指标的盘前 watchlist。
- **登录快捷方式**: 新增 `Login-Grit-DayTrading.cmd`，可双击启动本地后端、前端并打开复盘台，同时提供 backend/frontend helper cmd。
- **项目文档基线**: 建立 STP 日内交易闭环 Web 系统的产品范围、架构合同、技术计划和 agent 操作规则。
- **P0 Scaffold**: 建立 FastAPI、SQLite、STP TXT parser、导入 API、测试 fixture 和 React 复盘台骨架。
- **P0 多代理验收**: 补齐 storage contract、parser edge fixture、导入 API 幂等测试、复盘台 UI 和 DB/API/UI 一致性验收报告。
- **STP 证据账本合同**: 明确导入批次、原始文件 hash、原始行、解析版本、字段映射版本、账号归一和异常隔离要求。
- **成交市场上下文合同**: 明确富途行情只用于分钟线、VWAP、当日高低和成交量环境重建，行情缺失必须显式显示。
- **事实源边界**: 明确 STP TXT、富途行情和复盘结论只能互相引用，不能互相覆盖。

### 变更

- **首版优先级**: 将首版重心从单纯导入和报表提升为证据、成交和市场上下文的可追溯闭环。

### 修复

- **API 连接默认值**: 前端开发服务默认连接本项目后端，避免误连相邻服务导致复盘台 API 显示 404。
- **蜡烛图宽度**: 分钟蜡烛图会随复盘模块宽度自适应展开，不再在右侧留下大片空白。
- **复盘日期默认值**: 复盘台默认日期改为美东交易日口径，避免亚洲时区打开页面时提前切到下一天。
- **阶段提示**: 移除复盘台中与当前交付阶段不匹配的更新提示，避免把后续能力误认为已经上线。
- **无表头导入**: 无表头成交 TXT 可按默认字段合同解析，并把缺价格等问题留到具体异常行。
- **导入重解析**: 旧解析器造成的零行失败批次可在新解析器下重新解析，避免同一文件反复显示旧失败。
- **复盘统计**: 缺成交编号的重复成交行会逐行入账，交易股数按买卖配对数量展示。
- **复盘去重**: 同一批成交因补表头、编码或尾部空行修正后重导，成交列表和 KPI 不再重复计算重叠成交。
- **胜率和盈亏比**: PnL、胜率和盈亏比改为按每次 B&S 或 S&B 平仓 round-trip 计算，不再把同一标的整天合并成一笔。
- **失败语义**: 规定导入失败、解析失败、重复导入、行情缺失和时区错位都必须可见，不能静默通过。
- **缺成交编号幂等**: 缺少 execution id 的成交使用包含原始行 hash 的 fallback key，降低跨批次误合并风险。
